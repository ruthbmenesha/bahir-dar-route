    01
    HTTP Server Setup
    Starts web server on port 8080
    ▼
    Loads SWI-Prolog HTTP libraries and starts server. thread_get_message(_) keeps it alive.
    :- use_module(library(http/thread_httpd)).\nstart_server :- Port = 8080, http_server(http_dispatch, [port(Port)]), thread_get_message(_).
    02
    City Graph
    15 locations with GPS coordinates
    ▼
    Weighted undirected graph with location/3 and road/3 facts. Edge rule makes roads bidirectional.
    location('Bahir Dar University', 11.5942, 37.3886).\nroad('Bahir Dar University', 'Stadium', 1.2).
    03
    Haversine Heuristic
    Admissible h(n) for A*
    ▼
    Straight-line distance using Haversine formula — never overestimates, guaranteeing optimal A* paths.
    haversine(Lat1, Lon1, Lat2, Lon2, D) :- ... D is 6371 * C.
    04
    A* Algorithm
    f(n) = g(n) + h(n)
    ▼
    Open list stores f(F,G,Node,Path). Always expands lowest f. When goal is popped, path is found.
    astar(Start, Goal, Path, Cost) :- astar_loop([f(H,0,Start,[Start])], Goal, RevPath, Cost).
    05
    Alternative Routes
    Up to 3 different paths
    ▼
    Blocks one node from best path and re-runs A*, forcing a detour to generate alternatives.
    astar_avoid(Start, Goal, Avoid, Path, Cost).
    06
    JSON API
    Prolog → Browser response
    ▼
    reply_json/1 converts Prolog terms to JSON. CORS headers enable cross-origin requests from frontend.
    RESPONSE FORMAT
    { "status": "ok", "path": [...], "cost": 2.7, "coords": [...] }
